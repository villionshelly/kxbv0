# 合同管理模块 PRD

## 1. 产品目标

合同管理模块用于支持培训机构与家长围绕课程服务形成可查看、可确认、可下载、可归档的合同记录。模块需覆盖线下纸质合同照片管理与系统模板电子合同签约两类场景，并与学员购买记录联动，减少机构重复录入课程、金额、课时和服务周期。

## 2. 用户角色

### 2.1 机构端用户

机构管理员或具备合同管理权限的员工，可进行合同上传、合同发起、合同修改、合同下载、合同归档、盖章回传等操作。

### 2.2 家长端用户

已绑定学员的家长，可查看机构同步的纸质合同照片，查看待确认的电子合同内容，并完成电子合同确认。

## 3. 合同类型

### 3.1 纸质合同照片

纸质合同照片指机构将线下签署或线下留存的纸质合同拍照上传后，家长端可查看的合同记录。

纸质合同照片不生成 DOCX，不进入电子签约流程。

### 3.2 系统模板电子合同

系统模板电子合同指机构基于系统提供的课程服务协议模板，选择学员与购买记录后生成的电子合同。家长端确认后，合同状态变更为已签约。

系统模板电子合同需支持 DOCX 下载，机构可打印、盖章、拍照后进行盖章回传。

电子合同固定展示效力提示：

> 合同为电子合同有同等效力，如需盖章合同，请联系机构出具。

## 4. 信息架构

### 4.1 机构端入口

机构端个人中心的“合同管理”入口跳转至：

`/institution/contracts`

### 4.2 家长端入口

家长端合同管理入口跳转至：

`/parent/contracts`

### 4.3 学员购课记录入口

学员详情中的购买记录入口跳转至：

`/institution/students/{studentId}/purchase-records`

## 5. 数据模型

### 5.1 合同基础字段

| 字段 | 说明 |
| --- | --- |
| 合同 ID | 合同唯一标识 |
| 合同类型 | 纸质合同照片 / 系统模板电子合同 |
| 合同标题 | 机构设置的合同名称 |
| 机构名称 | 合同所属机构 |
| 学员 ID | 合同关联学员 |
| 学员姓名 | 合同关联学员姓名 |
| 家长姓名 | 合同关联家长姓名 |
| 创建时间 | 合同发起或上传时间 |
| 最近修改时间 | 合同被编辑后的时间 |
| 归档时间 | 合同归档时间 |
| 合同状态 | 根据合同类型和业务动作变化 |

### 5.2 纸质合同照片字段

| 字段 | 说明 |
| --- | --- |
| 照片列表 | 纸质合同照片，至少 1 张 |
| 状态 | 纸质已上传 |

### 5.3 系统模板电子合同字段

| 字段 | 说明 |
| --- | --- |
| 合同编号 | 系统按生成日期与合同 ID 自动生成 |
| 课程名称 | 来自购买记录或机构手动编辑 |
| 课包名称 | 来自购买记录或机构手动编辑 |
| 课时数量 | 来自购买记录或机构手动编辑 |
| 合同金额 | 来自购买记录或机构手动编辑 |
| 服务周期 | 默认来自购买记录的课包有效期 |
| 补充约定 | 机构填写的额外约定 |
| 生成日期 | 模板合同生成日期 |
| 家长确认时间 | 家长确认签约后的时间 |
| 盖章回传照片 | 机构打印盖章后上传的照片 |

### 5.4 购买记录字段

| 字段 | 说明 |
| --- | --- |
| 购买记录 ID | 购买记录唯一标识 |
| 学员 ID | 关联学员 |
| 购买日期 | 记录购买发生日期 |
| 课程名称 | 购买课程名称 |
| 课包名称 | 购买课包名称 |
| 付款金额 | 购买金额 |
| 总课时 | 课包总课时 |
| 已使用课时 | 已消耗课时 |
| 剩余课时 | 总课时减已使用课时 |
| 单课时价格 | 付款金额除以总课时 |
| 课包有效期 | 课程服务周期来源字段 |
| 状态 | 进行中 / 已完成 / 已退课 |
| 来源 | 手动录入 / 系统生成 |
| 备注 | 机构补充说明 |

## 6. 合同状态规则

### 6.1 纸质合同照片状态

纸质合同照片上传后状态为“纸质已上传”。

纸质合同照片可修改、归档、下载照片。

纸质合同照片不支持家长确认签约。

### 6.2 系统模板电子合同状态

| 状态 | 触发条件 |
| --- | --- |
| 待家长签约 | 机构发起模板合同后 |
| 已签约 | 家长点击确认签约后 |
| 已盖章回传 | 机构上传盖章合同照片后 |
| 已归档 | 机构归档合同后 |

### 6.3 状态变更限制

1. 待签约合同可修改，修改后仍为待家长签约。
2. 已签约合同可修改，修改后状态重置为待家长签约，并清空原家长确认时间。
3. 已盖章回传合同可修改，修改后状态重置为待家长签约，并清空原盖章回传照片。
4. 待签约模板合同可删除。
5. 已签约和已盖章回传模板合同不支持删除。
6. 已归档合同不在家长端展示。
7. 已归档合同在机构端保留记录，可查看和下载，不再进行修改、删除、盖章回传等业务操作。

## 7. 机构端功能需求

### 7.1 合同列表

机构端合同列表需展示机构全部合同，包括未归档合同和已归档合同。

列表卡片保持精简，展示以下信息：

| 信息 | 说明 |
| --- | --- |
| 合同标题 | 合同名称 |
| 合同类型 | 纸质合同 / 电子合同 |
| 学员姓名 | 关联学员 |
| 家长姓名 | 关联家长 |
| 合同状态 | 当前状态 |
| 创建时间 | 发起或上传时间 |
| 最近修改时间 | 有修改时展示 |
| 归档时间 | 已归档时展示 |

列表卡片不展示合同正文摘要。

### 7.2 合同搜索

合同列表需支持搜索。搜索范围包括：

1. 合同标题
2. 学员姓名
3. 家长姓名
4. 机构名称
5. 合同类型
6. 合同状态
7. 课程名称
8. 课包名称

搜索结果为空时，展示空状态提示。

### 7.3 上传纸质合同

机构可创建纸质合同照片记录。

必填字段：

1. 学员
2. 合同标题
3. 合同照片

纸质合同创建后同步至家长端，家长端可查看照片。

### 7.4 发起模板合同

机构可创建系统模板电子合同。

创建流程：

1. 输入学员姓名、家长姓名或课程关键词搜索学员。
2. 选择目标学员。
3. 系统展示该学员购买记录，进行中的购买记录排在前面。
4. 选择购买记录后，系统自动回填合同字段。
5. 机构可手动调整合同标题、课程名称、课包名称、课时数量、金额、服务周期和补充约定。
6. 机构确认发起签约后，家长端显示待签约合同。

自动回填规则：

| 合同字段 | 来源 |
| --- | --- |
| 课程名称 | 购买记录课程名称 |
| 课包名称 | 购买记录课包名称 |
| 课时数量 | 购买记录总课时 |
| 金额 | 购买记录付款金额 |
| 服务周期 | 购买记录课包有效期 |

### 7.5 修改合同

纸质合同支持修改合同标题和关联学员。

模板合同支持修改以下字段：

1. 合同标题
2. 关联学员
3. 课程名称
4. 课包名称
5. 课时数量
6. 合同金额
7. 服务周期
8. 补充约定

模板合同每次修改后，合同状态重置为待家长签约。家长需重新确认。

### 7.6 删除合同

仅待签约的模板合同支持删除。

已签约、已盖章回传、纸质合同照片不支持删除。

### 7.7 归档合同

机构可归档合同。

归档后：

1. 机构端仍可在合同列表中查看归档记录。
2. 家长端不展示归档合同。
3. 合同不再支持修改、删除、盖章回传。
4. 模板合同仍可查看正文和下载 DOCX。
5. 纸质合同仍可查看和下载照片。

### 7.8 查看合同正文

模板合同支持查看完整合同正文。

合同正文需包含：

1. 合同标题
2. 合同编号
3. 甲方机构名称
4. 甲方联系人
5. 乙方家长姓名
6. 学员姓名
7. 生成日期
8. 确认时间
9. 课程服务内容
10. 费用及支付
11. 排课、请假与课时扣减
12. 服务周期与延期
13. 退费、转课与变更
14. 双方权利义务
15. 补充约定
16. 电子确认与合同效力
17. 争议解决
18. 签署确认信息

合同编号只在标题下方展示，不在甲乙方信息区重复展示。

甲方联系人展示在甲方下面。

学员姓名展示在乙方下面。

签约状态在合同底部独立成行展示。

正文中由系统填充的字段需加粗并加下划线展示。

### 7.9 下载 DOCX

系统模板电子合同支持下载 DOCX 文件。

文件命名规则：

`{学员名}-{合同标题}-{YYYYMMDD}.docx`

DOCX 内容需与页面合同正文保持一致，包含合同正文、签署确认信息、电子合同效力提示。

纸质合同照片不生成 DOCX。

### 7.10 盖章回传

模板合同支持机构下载 DOCX 后打印盖章。

机构可将盖章后的合同照片上传回传。

回传后合同状态变更为“已盖章回传”，家长端可查看盖章回传照片。

## 8. 家长端功能需求

### 8.1 合同列表

家长端合同列表展示与当前家长绑定学员相关的未归档合同。

列表展示：

1. 合同标题
2. 合同类型
3. 机构名称
4. 学员姓名
5. 合同状态
6. 创建时间或确认时间

### 8.2 查看纸质合同照片

家长点击纸质合同后，进入照片查看弹层。

家长可查看合同照片，并可下载照片。

### 8.3 查看模板电子合同

家长点击模板电子合同后，进入合同详情弹层。

合同详情展示完整合同正文和电子合同效力提示。

### 8.4 确认签约

待签约模板合同展示“确认签约”按钮。

家长点击确认后：

1. 系统记录确认时间。
2. 合同状态变更为已签约。
3. 机构端同步展示已签约状态。

已签约或已盖章回传合同不再展示确认签约按钮。

### 8.5 查看盖章回传照片

机构上传盖章回传照片后，家长端合同详情展示盖章合同照片。

## 9. 购买记录功能需求

### 9.1 添加购买记录

机构在学员购买记录页面可添加购买记录。

必填字段：

1. 购买日期
2. 课程
3. 付款金额
4. 课时数量
5. 课包有效期

选填字段：

1. 已使用课时
2. 记录状态
3. 备注

### 9.2 课包有效期

课包有效期用于描述本次购买课包对应的服务周期。

该字段在创建模板合同时自动提取为合同的服务周期。

### 9.3 购买记录排序

购买记录在合同创建流程中展示时，排序规则为：

1. 进行中的购买记录排在前面。
2. 同状态下按购买日期倒序排列。

### 9.4 购买记录联动合同

选择购买记录后，合同创建表单自动回填课程名称、课包名称、课时数量、金额和服务周期。

机构可在回填后手动调整字段。

## 10. 合同模板正文要求

### 10.1 头部信息

合同正文头部展示：

1. 合同标题
2. 合同编号

合同编号位于标题下方第二行。

### 10.2 甲乙方信息

甲乙方信息展示顺序：

1. 甲方（培训机构）
2. 甲方联系人
3. 乙方（家长/监护人）
4. 学员姓名
5. 生成日期
6. 确认时间

### 10.3 正文条款

合同正文至少包含以下条款：

1. 课程服务内容
2. 费用及支付
3. 排课、请假与课时扣减
4. 服务周期与延期
5. 退费、转课与变更
6. 双方权利义务
7. 补充约定
8. 电子确认与合同效力
9. 争议解决

### 10.4 字段样式

合同正文中的动态填充字段需加粗并加下划线展示。

动态字段包括：

1. 课程名称
2. 课包名称
3. 课时数量
4. 合同金额
5. 服务周期
6. 补充约定

### 10.5 底部签署确认

底部签署确认展示：

1. 甲方
2. 甲方联系人
3. 乙方
4. 学员
5. 签约状态

签约状态需独立成行。

## 11. 权限与限制

1. 只有机构端用户可创建、修改、归档、删除、下载、盖章回传合同。
2. 家长端用户只能查看合同、下载照片、确认待签约模板合同。
3. 家长端不能修改合同内容。
4. 已归档合同不展示给家长。
5. 已签约合同不支持删除。
6. 纸质合同照片不参与电子签约状态流转。
7. 模板电子合同不接入第三方电子签平台，家长点击确认签约即完成系统内确认。

## 12. 验收标准

### 12.1 机构端合同管理

1. 机构可进入合同管理页面。
2. 合同列表可按合同、学员、家长、课程、状态搜索。
3. 合同列表卡片展示精简信息，不展示合同正文摘要。
4. 机构可上传纸质合同照片。
5. 机构可发起模板电子合同。
6. 机构可修改纸质合同。
7. 机构可修改模板合同，修改后状态变为待家长签约。
8. 待签约模板合同可删除。
9. 已签约模板合同不可删除。
10. 合同可归档，归档后家长端不展示。
11. 模板合同可查看完整正文。
12. 模板合同可下载 DOCX。
13. 模板合同可上传盖章回传照片。

### 12.2 学员购买记录

1. 添加购买记录时必须填写课包有效期。
2. 购买记录卡片展示课包有效期。
3. 合同创建时可选择学员购买记录。
4. 进行中的购买记录排在前面。
5. 选择购买记录后自动回填课程名称、课包名称、课时数量、金额、服务周期。

### 12.3 家长端合同管理

1. 家长端展示未归档合同。
2. 家长可查看纸质合同照片。
3. 家长可查看模板电子合同正文。
4. 待签约模板合同可确认签约。
5. 确认签约后记录确认时间，机构端状态同步为已签约。
6. 已盖章回传合同在家长端展示盖章照片。

### 12.4 合同正文与 DOCX

1. 合同编号只在标题下第二行展示。
2. 甲方下面展示甲方联系人。
3. 乙方下面展示学员姓名。
4. 正文字段填充内容加粗并加下划线。
5. 底部展示甲方联系人。
6. 学员展示在乙方下面。
7. 签约状态独立成行。
8. DOCX 内容与页面合同正文一致。

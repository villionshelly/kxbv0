## 一、系统概述

### 1.1 产品架构

课小宝采用三端架构设计：

```

                                │
        ┌───────────────────────┴───────────────────────┐
        ▼                                               ▼
┌───────────────────┐                       ┌───────────────────┐
│   机构端 (B端)     │ ◄─── 绑定/同步 ───► │   家长端 (C端)     │
│  - 管理员视图      │                       │  - 课表管理        │
│  - 教师视图        │                       │  - 消课记录        │
│  - 课程/班次管理   │                       │  - 自主记账        │
│  - 学员管理        │                       │  - 成长档案        │
│  - AI功能         │                       │  - AI助理          │
└───────────────────┘                       └───────────────────┘
```

## 三、业务流程详解

### 3.1 课程类型与消课来源

系统支持两种课程类型：

| 类型 | courseType | 说明 | 消课方式 | 请假/调休 |
|------|-----------|------|---------|----------|
| 机构课 | `institution` | 与培训机构绑定的正式课程 | 机构老师消课，数据同步到家长端 | 需机构/老师审批 |
| 自主记账 | `self` | 家长自行添加管理的课程 | 家长自行记录消课 | 系统自动同意 |

## 四、核心场景流程

### 4.1 场景一：家长端自主记账全流程

> **场景描述**：家长想要记录孩子在私教老师/非平台机构上课的课时消耗情况

#### 流程步骤

```
┌─────────────────────────────────────────────────────────────────┐
│ 步骤1：添加自主课程                                              │
├─────────────────────────────────────────────────────────────────┤
│ 入口：首页 → 添加课程 → 选择"自主记账"                           │
│                                                                 │
│ 填写信息：                                                       │
│ - 课程名称：跆拳道 *                                             │
│ - 课程分类：体育运动 *                                           │
│ - 总课时数：60 *                                                 │
│ - 总价：4800元 *                                                 │
│ - 机构/老师名称：龙武跆拳道馆（选填）                             │
│ - 上课时间：周日 16:00-17:30                                     │
│                                                                 │
│ 结果：创建 courseType='self' 的课程，自动生成周期性排课          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 步骤2：课表展示                                                  │
├─────────────────────────────────────────────────────────────────┤
│ 自主记账课程与机构课程在日历中统一展示                            │
│ 区分标识：课程卡片右上角显示"自主"标签                           │
│                                                                 │
│ 示例：                                                          │
│ ┌──────────────────────────────────┐                           │
│ │ 周日 16:00-17:30                 │                           │
│ │ 跆拳道            [自主]         │                           │
│ │ 龙武跆拳道馆 · 陈教练            │                           │
│ └──────────────────────────────────┘                           │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 步骤3：自主消课                                                  │
├─────────────────────────────────────────────────────────────────┤
│ 入口：课表 → 点击课程 → "确认上课"                               │
│                                                                 │
│ 操作：                                                          │
│ - 确认日期和时间                                                 │
│ - 可添加备注（如：品势练习）                                      │
│ - 点击"确认消课"                                                │
│                                                                 │
│ 结果：                                                          │
│ - 生成消课记录 (source='self', status='attended')               │
│ - remainingClasses 减 1                                         │
│ - 记录同步到消课历史                                             │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 步骤4：请假/调休（自动同意）                                      │
├─────────────────────────────────────────────────────────────────┤
│ 入口：课程详情 → 点击某节课 → "请假/调休"                         │
│                                                                 │
│ 请假流程：                                                       │
│ - 选择原因（可选）                                               │
│ - 提交后【系统自动同意】                                         │
│ - 提示："自主记账课程，系统已自动同意，课时暂不扣除"              │
│                                                                 │
│ 调休流程：                                                       │
│ - 选择补课日期                                                   │
│ - 提交后【系统自动同意】                                         │
│ - 生成调休补课记录 (status='makeup')                            │
│                                                                 │
│ ⚠️ 与机构课的区别：                                              │
│ - 机构课需等待老师/机构审批                                       │
│ - 自主记账课程无需审批，即时生效                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### 数据流示例

```javascript
// 1. 创建自主记账课程

  remainingClasses: 60,
  price: 4800,  // 总价
  courseType: 'self'  // 关键标识

// 2. 生成排课（每周日16:00）
const scheduleItem = {
  courseId: '4',
  courseName: '跆拳道',
  date: '2026-05-24',  // 下个周日
  startTime: '16:00',
  endTime: '17:30',
  courseType: 'self'
}

// 3. 自主消课记录
const classRecord = {
  courseId: '4',
  courseName: '跆拳道',
  date: '2026-05-17',
  deduction: 1,
  source: 'self',      // 消课来源：自主
  note: '品势练习',

// 4. 请假（自动同意）

  courseId: '4',
  date: '2026-05-24',
  deduction: 0,        // 请假不扣课时

  status: 'leave',
  // 无需 approvedBy，系统自动同意
}
```

### 4.3 场景三：机构自主开通，邀请家长成为新学员

#### 流程图

## 五、请假与调休业务规则

### 5.1 请假规则对比

| 维度 | 机构课 (institution) | 自主记账 (self) |
|------|---------------------|----------------|
| 发起入口 | 课程详情 → 请假/调休 | 课程详情 → 请假/调休 |
| 审批流程 | 需老师/机构审批 | 系统自动同意 |
| 审批时效 | 通常24小时内 | 即时生效 |
| 课时处理 | 暂不扣除，待审批结果 | 直接不扣除 |
| 通知对象 | 通知机构老师 | 无需通知 |
| 状态流转 | pending → approved/rejected | 直接 approved |

### 5.2 调休规则对比

| 维度 | 机构课 (institution) | 自主记账 (self) |
|------|---------------------|----------------|
| 补课选择 | 机构提供可选时段 | 家长自选任意时间 |
| 审批流程 | 需老师/机构确认补课时段 | 系统自动确认 |
| 补课安排 | 机构统一安排 | 家长自行安排 |
| 课时处理 | 原课请假不扣，补课正常扣 | 原课不扣，补课扣1 |

## 八、数据同步机制

### 8.1 机构 → 家长 数据同步

当机构端发生以下操作时，自动同步到关联的家长端：

| 机构端操作 | 同步内容 | 同步时机 |
|-----------|---------|---------|
| 消课 | 消课记录、剩余课时 | 实时 |
| 修改排课 | 课表时间变更 | 实时 |
| 审批请假 | 请假状态、调休安排 | 实时 |
| 发送成长报告 | 报告内容 | 实时 |
| 上传课堂照片 | 成长档案照片 | 实时 |

### 8.2 同步实现示例

```typescript
// 机构消课后触发同步
async function onInstitutionCheckin(checkinData) {
  // 1. 保存机构端消课记录
  await saveCheckinRecord(checkinData)
  
  // 2. 查找关联的家长账号
  const student = await getStudent(checkinData.studentId)
  const parentBinding = await getParentBinding(student.parentPhone)
  
  if (parentBinding && parentBinding.status === 'bindbindedbound') {
    // 3. 同步到家长端
    await createParentClassRecord({
      courseId: parentBinding.linkedCourseId,
      date: checkinData.date,
      source: 'institution',
      status: checkinData.status,
      note: checkinData.note
    })
    
    // 4. 更新家长端课时
    await updateParentCourseRemaining(
      parentBinding.linkedCourseId,
      -checkinData.deduction
    )
    
    // 5. 发送通知（可选）
    await sendParentNotification(parentBinding.parentId, {
      type: 'checkin',
      message: `${student.name}的${checkinData.courseName}已消课`
    })
  }
}
```
